System description

Pharmacy Management System Overview

The pharmacy management system is a comprehensive platform designed to support pharmacy operations, with modules tailored for owners, pharmacists, and customers. Below is a summary of the system’s key functionalities, based on the provided user stories.

1. Financial Reports and Analysis Module

The pharmacy owner has access to detailed financial reports, enabling revenue and cost monitoring. The system generates sales reports, analyzes costs related to supplier collaboration, and compares contractual conditions. It also allows the review of transaction histories with wholesalers and suppliers.

2. Real-Time Inventory Monitoring

The system tracks inventory levels in real-time and automatically alerts when supplies need replenishing. The owner can set minimum and maximum stock levels, supporting effective inventory management.

3. Personnel Management

This module provides tools for scheduling and tracking employee performance, enhancing team management and operational efficiency.

4. Regulatory Compliance Management

The operations module ensures pharmacy activities comply with current pharmaceutical regulations, provides reminders for required inspections, and securely stores documentation necessary for regulatory audits.

5. Promotions and Marketing

The owner can create and manage promotions to boost sales. The system also collects customer feedback and analyzes purchasing preferences, helping tailor offerings to market needs.

6. Customer Interactions

The system allows customers to view available over-the-counter medications, check the availability of prescription drugs, and fulfill e-prescriptions online. Customers can also ask pharmacists questions and receive reminders about expiring prescriptions.

7. Online Orders and Payments

Customers can place orders online with secure, easy payment options. The system provides order histories and generates electronic invoices for convenience.

8. Pharmacist Support

Pharmacists can directly order missing medications from suppliers, manage customer offerings, process prescriptions, and generate sales reports. The module also tracks medication expiration dates and provides access to a drug interaction database.

The pharmacy management system integrates with pharmaceutical wholesalers and supports tracking customer purchases, helping pharmacists maintain continuity of care and align product offerings with customer needs.

Singleton

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system needs to ensure that the system configuration is globally available as a single instance.

Solution
The Singleton pattern allows the creation of a class where exactly one instance is available throughout the system.

Diagram klas

Kod wzorca

public class Configuration {
	private static Configuration instance;
	private Map settings;

	private Configuration() {
		settings = new HashMap<>();
	}

	public static synchronized Configuration getInstance() {
		if (instance == null) {
			instance = new Configuration();
		}
		return instance;
	}

	public String getSetting(String key) {
		return settings.get(key);
	}

	public void setSetting(String key, String value) {
		settings.put(key, value);
	}
}
Kod testowy wzorca

public class ConfigurationTest {
	public static void main(String[] args) {
		Configuration config = Configuration.getInstance();
		config.setSetting("language", "EN");
		System.out.println(config.getSetting("language")); // Outputs: EN
	}
}
Metoda fabrykująca

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system must allow creating different types of notifications (SMS, email) in a way that facilitates extension.

Solution
The Factory Method defines an interface for creating objects but allows subclasses to decide which specific object to create.

Diagram klas

Kod wzorca

public interface Notification {
    void send(String message);
}

public class SmsNotification implements Notification {
    public void send(String message) {
        System.out.println("SMS: " + message);
    }
}

public class EmailNotification implements Notification {
    public void send(String message) {
        System.out.println("Email: " + message);
    }
}

public interface NotificationFactory {
    Notification createNotification();
}

public class SmsNotificationFactory implements NotificationFactory {
    public Notification createNotification() {
        return new SmsNotification();
    }
}

public class EmailNotificationFactory implements NotificationFactory {
    public Notification createNotification() {
        return new EmailNotification();
    }
}
Kod testowy wzorca

public class NotificationFactoryTest {
    public static void main(String[] args) {
        NotificationFactory factory = new SmsNotificationFactory();
        Notification notification = factory.createNotification();
        notification.send("Test message"); // Outputs: SMS: Test message
    }
}

Budowniczy

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system needs to create complex orders containing multiple medication items in a flexible way.

Solution
The Builder pattern allows the step-by-step construction of complex objects, enabling diverse configurations.

Diagram klas

Kod wzorca

public class Order {
    private List medicines;
    private Customer customer;

    public Order(List medicines, Customer customer) {
        this.medicines = medicines;
        this.customer = customer;
    }

    public void displayDetails() {
        System.out.println("Order for customer: " + customer.getName());
        for (Medicine medicine : medicines) {
            System.out.println("- " + medicine.getName());
        }
    }
}

public class OrderBuilder {
    private List medicines = new ArrayList<>();
    private Customer customer;

    public OrderBuilder addMedicine(Medicine medicine) {
        medicines.add(medicine);
        return this;
    }

    public OrderBuilder setCustomer(Customer customer) {
        this.customer = customer;
        return this;
    }

    public Order build() {
        return new Order(medicines, customer);
    }
}

                .setCustomer(customer)

Prototyp

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system needs to create copies of existing medications to manage their various configurations within orders.

Solution
The Prototype pattern allows the creation of new objects by copying existing instances.

Diagram klas

Kod wzorca

public class Medicine implements Cloneable {
    private String name;
    private double price;

    public Medicine(String name, double price) {
        this.name = name;
        this.price = price;
    }

    @Override
    public Medicine clone() {
        return new Medicine(this.name, this.price);
    }

    @Override
    public String toString() {
        return "Medicine{name='" + name + "', price=" + price + "}";
    }
}
Kod testowy wzorca

public class PrototypeTest {
    public static void main(String[] args) {
        Medicine original = new Medicine("Ibuprofen", 10.99);
        Medicine copy = original.clone();

        System.out.println(original); // Outputs: Medicine{name='Ibuprofen', price=10.99}
        System.out.println(copy);    // Outputs: Medicine{name='Ibuprofen', price=10.99}
    }
}
Adapter

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system uses an external supplier module that has a different interface than required by the application.

Solution
The Adapter pattern allows adjusting the class interface to meet the client's requirements without changing the existing code.

Diagram klas

Kod wzorca

public interface SupplierAPI {
    void requestMedicine(String name, int quantity);
}

public class ExternalSupplier {
    public void makeOrder(String medicineName, int qty) {
        System.out.println("Order placed for " + qty + " of " + medicineName);
    }
}

public class SupplierAdapter implements SupplierAPI {
    private ExternalSupplier supplier;

    public SupplierAdapter(ExternalSupplier supplier) {
        this.supplier = supplier;
    }

    @Override
    public void requestMedicine(String name, int quantity) {
        supplier.makeOrder(name, quantity);
    }
}
Kod testowy wzorca

public class AdapterTest {
    public static void main(String[] args) {
        ExternalSupplier externalSupplier = new ExternalSupplier();
        SupplierAPI supplierAPI = new SupplierAdapter(externalSupplier);

        supplierAPI.requestMedicine("Aspirin", 100);
    }
}

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system must handle different types of reports (e.g., sales and inventory) in various formats (e.g., PDF, CSV).

Solution
The Bridge pattern separates abstraction from its implementation, allowing independent changes.

Diagram klas

Kod wzorca

public abstract class Report {
    protected ReportFormatter formatter;

    public Report(ReportFormatter formatter) {
        this.formatter = formatter;
    }

    public abstract void generate();
}

public interface ReportFormatter {
    String format(String data);
}

public class PDFFormatter implements ReportFormatter {
    @Override
    public String format(String data) {
        return "PDF formatted: " + data;
    }
}

public class CSVFormatter implements ReportFormatter {
    @Override
    public String format(String data) {
        return "CSV formatted: " + data;
    }
}

public class SalesReport extends Report {
    public SalesReport(ReportFormatter formatter) {
        super(formatter);
    }

    @Override
    public void generate() {
        System.out.println(formatter.format("Sales Report Data"));
    }
}

public class StockReport extends Report {
    public StockReport(ReportFormatter formatter) {
        super(formatter);
    }

    @Override
    public void generate() {
        System.out.println(formatter.format("Stock Report Data"));
    }
}
Kod testowy wzorca

public class BridgeTest {
    public static void main(String[] args) {
        Report salesReport = new SalesReport(new PDFFormatter());
        salesReport.generate();

        Report stockReport = new StockReport(new CSVFormatter());
        stockReport.generate();
    }
}

Diagram klas

Kod wzorca

public abstract class Mediator {
    public abstract void send(String message, Colleague colleague);
}

public abstract class Colleague {
    protected Mediator mediator;

    public Colleague(Mediator mediator) {
        this.mediator = mediator;
    }

    public abstract void send(String message);
    public abstract void receive(String message);
}

public class OrderMediator extends Mediator {
    private List colleagues = new ArrayList<>();

    public void registerColleague(Colleague colleague) {
        colleagues.add(colleague);
    }

    @Override
    public void send(String message, Colleague sender) {
        for (Colleague colleague : colleagues) {
            if (colleague != sender) {
                colleague.receive(message);
            }
        }
    }
}

        super(mediator);
    }

    @Override
    public void send(String message) {
        System.out.println("Pharmacist sends: " + message);
        mediator.send(message, this);
    }

    @Override
    public void receive(String message) {
        System.out.println("Pharmacist receives: " + message);
    }
}

public class Warehouse extends Colleague {
    public Warehouse(Mediator mediator) {
        super(mediator);
    }

    @Override
    public void send(String message) {
        System.out.println("Warehouse sends: " + message);
        mediator.send(message, this);
    }

    @Override
    public void receive(String message) {
        System.out.println("Warehouse receives: " + message);
    }
}

public class Order {
    private String state;

    public void setState(String state) {
        this.state = state;
    }

    public String getState() {
        return state;
    }

    public OrderMemento createMemento() {
        return new OrderMemento(state);
    }

    public void restore(OrderMemento memento) {
        this.state = memento.getState();
    }
}

public class OrderMemento {
    private final String state;

    public OrderMemento(String state) {
        this.state = state;
    }

    public String getState() {
        return state;
    }
}

import java.util.Stack;

public class OrderManager {
    private Stack history = new Stack<>();

    public void saveOrder(Order order) {
        history.push(order.createMemento());
    }

    public void undo(Order order) {
        if (!history.isEmpty()) {
            order.restore(history.pop());
        }
    }
}

Iterator

Autor wzorca: Artur Szymkiewicz

Problem
The pharmacy system needs to allow iterating over a collection of medications in a way that is independent of the implementation of the collection.

Solution
The Iterator pattern allows sequentially traversing the elements of a collection without revealing its internal structure.

Diagram klas

Kod wzorca

import java.util.ArrayList;
import java.util.List;

public interface Iterator {
    boolean hasNext();
    T next();
}

public interface Aggregate {
    Iterator createIterator();
}

public class Medicine {
    private String name;

    public Medicine(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

public class MedicineCollection implements Aggregate {
    private List medicines = new ArrayList<>();

    public void addMedicine(Medicine medicine) {
        medicines.add(medicine);
    }

    @Override
    public Iterator createIterator() {
        return new MedicineIterator(medicines);
    }
}

public class MedicineIterator implements Iterator {
    private List medicines;
    private int position = 0;

    public MedicineIterator(List medicines) {
        this.medicines = medicines;
    }

    @Override
    public boolean hasNext() {
        return position < medicines.size();
    }

    @Override
    public Medicine next() {
        if (!hasNext()) {
            throw new IllegalStateException("No more elements");
        }
        return medicines.get(position++);
    }
}
Kod testowy wzorca

public class IteratorTest {
    public static void main(String[] args) {
        MedicineCollection collection = new MedicineCollection();
        collection.addMedicine(new Medicine("Aspirin"));
        collection.addMedicine(new Medicine("Vitamin C"));
        collection.addMedicine(new Medicine("Ibuprofen"));

        Iterator iterator = collection.createIterator();
        while (iterator.hasNext()) {
            System.out.println("Medicine: " + iterator.next().getName());
        }
        // Outputs:
        // Medicine: Aspirin
        // Medicine: Vitamin C
        // Medicine: Ibuprofen
    }
}
